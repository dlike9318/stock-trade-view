# 云平台部署计划

## 部署架构

```
GitHub 仓库
    ├── backend/     → Railway (后端服务 + WebSocket)
    ├── frontend/    → Vercel (前端应用)
    ├── admin/       → Vercel (管理端系统)
    └── miniprogram/ → 微信开发者工具上传
```

## 推荐的免费云平台

| 服务 | 平台 | 免费额度 | 特点 |
|------|------|----------|------|
| 后端服务 | Railway | 500小时/月 | 支持WebSocket，自动部署 |
| 前端应用 | Vercel | 无限制 | 速度快，支持自定义域名 |
| 管理端系统 | Vercel | 无限制 | 与前端同一账号管理 |

## 部署步骤

### 1. 准备工作
- 创建 GitHub 账号（如果没有）
- 创建 Vercel 账号（使用 GitHub 登录）
- 创建 Railway 账号（使用 GitHub 登录）

### 2. 代码调整
- 添加环境变量配置
- 创建部署配置文件
- 修改硬编码地址为环境变量

### 3. 部署后端到 Railway
- 连接 GitHub 仓库
- 配置环境变量
- 自动部署

### 4. 部署前端到 Vercel
- 连接 GitHub 仓库
- 配置环境变量（后端API地址）
- 自动部署

### 5. 部署管理端到 Vercel
- 同前端部署流程

### 6. 微信小程序配置
- 修改 API 地址为 Railway 后端地址
- 在微信开发者工具中上传代码

## 需要创建的配置文件

1. `railway.json` - Railway 部署配置
2. `vercel.json` - Vercel 部署配置
3. `.env.example` - 环境变量示例文件
4. 修改各模块使用环境变量

## 预计部署时间
- 准备工作：10分钟
- 后端部署：5分钟
- 前端部署：5分钟
- 管理端部署：5分钟
- 小程序配置：5分钟
- **总计：约30分钟**

## 部署后访问地址
- 后端API：`https://your-app.railway.app`
- 前端应用：`https://your-app.vercel.app`
- 管理端系统：`https://your-admin.vercel.app`
